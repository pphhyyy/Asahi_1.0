# 🔐 权限与安全详解

## Android 权限系统概述

Android 采用**权限模型**来保护用户隐私和设备安全。App 需要什么能力就必须声明并获得对应权限，否则系统会拒绝操作。

---

## 本项目使用的权限

### 权限总览

| 权限 | 级别 | 获取方式 | 用途 |
|------|------|---------|------|
| `INTERNET` | 普通 | 安装时自动授予 | 网络请求（调用翻译 API）|
| `FOREGROUND_SERVICE` | 普通 | 安装时自动授予 | 运行前台服务 |
| `FOREGROUND_SERVICE_MEDIA_PROJECTION` | 普通 | 安装时自动授予 | 使用截屏功能的前台服务 |
| `POST_NOTIFICATIONS` | 危险 | 运行时弹窗申请 | 显示通知（Android 13+）|
| `SYSTEM_ALERT_WINDOW` | 特殊 | 跳转系统设置 | 显示悬浮窗 |
| `BIND_ACCESSIBILITY_SERVICE` | 签名 | 系统自动绑定 | 无障碍服务专用 |

---

## 权限的三个级别

### 1. 普通权限（Normal Permissions）

**不需要用户同意**，在 AndroidManifest.xml 中声明即可。系统在安装时自动授予。

```xml
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.FOREGROUND_SERVICE" />
```

为什么 INTERNET 是普通权限？因为绝大多数 App 都需要联网，而且单纯的网络访问对用户隐私影响较小。

### 2. 危险权限（Dangerous Permissions）

**需要用户明确同意**。App 运行时弹出对话框请求，用户可以拒绝。

```kotlin
// 检查权限
if (ContextCompat.checkSelfPermission(this, Manifest.permission.POST_NOTIFICATIONS)
    != PackageManager.PERMISSION_GRANTED) {
    // 请求权限
    ActivityCompat.requestPermissions(
        this,
        arrayOf(Manifest.permission.POST_NOTIFICATIONS),
        REQUEST_NOTIFICATION_PERMISSION
    )
}
```

用户会看到：
```
┌─────────────────────────────┐
│  允许"实时翻译"向您发送通知？  │
│                             │
│   [ 拒绝 ]    [ 允许 ]       │
└─────────────────────────────┘
```

### 3. 特殊权限（Special Permissions）

无法通过普通方式申请，**必须引导用户到系统设置页面手动开启**。

#### `SYSTEM_ALERT_WINDOW`（悬浮窗权限）

```kotlin
// 检查是否已授权
if (!Settings.canDrawOverlays(this)) {
    // 跳转到系统设置页面
    val intent = Intent(
        Settings.ACTION_MANAGE_OVERLAY_PERMISSION,
        Uri.parse("package:$packageName")
    )
    startActivityForResult(intent, REQUEST_OVERLAY_PERMISSION)
}
```

用户会看到系统设置页面：
```
┌─────────────────────────────┐
│  悬浮窗权限                   │
│                             │
│  实时翻译                     │
│  [ 允许显示在其他应用上层 ]     │
│      ○ 关 / ● 开            │
└─────────────────────────────┘
```

## 屏幕截图权限 —— MediaProjection

屏幕截图不是通过传统权限系统获取的，而是通过 **MediaProjection API**：

```kotlin
// 请求截屏权限
val projectionManager = getSystemService(MEDIA_PROJECTION_SERVICE) as MediaProjectionManager
startActivityForResult(
    projectionManager.createScreenCaptureIntent(),
    REQUEST_MEDIA_PROJECTION
)
```

系统会弹出确认对话框：
```
┌──────────────────────────────────┐
│  开始录制或投射屏幕？               │
│                                  │
│  "实时翻译"将会开始捕获屏幕上       │
│  显示的所有内容。                   │
│                                  │
│    [ 取消 ]    [ 立即开始 ]        │
└──────────────────────────────────┘
```

#### 授权数据的传递

截屏授权在 MainActivity 中获取，但实际使用在 OverlayService 中。传递方式：

```kotlin
// MainActivity 中获取
override fun onActivityResult(requestCode: Int, resultCode: Int, data: Intent?) {
    if (requestCode == REQUEST_MEDIA_PROJECTION) {
        // 保存到静态变量
        pendingResultCode = resultCode
        pendingResultData = data?.clone() as Intent
        // 启动服务
        startOverlayService()
    }
}

// OverlayService 中使用
override fun onStartCommand(intent: Intent?, flags: Int, startId: Int): Int {
    val resultCode = MainActivity.pendingResultCode
    val resultData = MainActivity.pendingResultData
    // 清除静态变量
    MainActivity.pendingResultCode = Activity.RESULT_CANCELED
    MainActivity.pendingResultData = null
    
    // 创建 MediaProjection
    mediaProjection = projectionManager.getMediaProjection(resultCode, resultData!!)
}
```

> ⚠️ Android 14（API 34）要求 MediaProjection 必须在**前台服务启动后**才能创建，这就是为什么先 `startForeground()` 再 `getMediaProjection()`。

## 无障碍服务权限

无障碍服务（AccessibilityService）是最特殊的，它不通过权限系统授权，而是需要用户：

1. 打开系统 **设置 → 无障碍**
2. 找到 "实时翻译" 服务
3. 手动开启

```kotlin
// 引导用户打开设置
startActivity(Intent(Settings.ACTION_ACCESSIBILITY_SETTINGS))
```

这样设计的原因是无障碍服务权限**极高**（可以读取所有屏幕内容），必须由用户明确同意。

## 前台服务通知

Android 8.0 起，前台服务**必须**显示通知。这是为了让用户知道有 App 在后台运行。

### 创建通知渠道

Android 8.0+ 要求通知必须关联到一个**通知渠道**：

```kotlin
private fun createNotificationChannel() {
    val channel = NotificationChannel(
        CHANNEL_ID,                           // 渠道 ID
        getString(R.string.notification_channel), // 渠道名称
        NotificationManager.IMPORTANCE_LOW    // 重要级别（低 = 不发声）
    )
    val manager = getSystemService(NotificationManager::class.java)
    manager.createNotificationChannel(channel)
}
```

### 创建通知

```kotlin
private fun createNotification(): Notification {
    val pendingIntent = PendingIntent.getActivity(
        this, 0,
        Intent(this, MainActivity::class.java),
        PendingIntent.FLAG_UPDATE_CURRENT or PendingIntent.FLAG_IMMUTABLE
    )

    return Notification.Builder(this, CHANNEL_ID)
        .setContentTitle("实时翻译运行中")
        .setContentText("点击返回应用")
        .setSmallIcon(android.R.drawable.ic_menu_edit)
        .setContentIntent(pendingIntent)
        .build()
}
```

| 要素 | 含义 |
|------|------|
| `PendingIntent` | 点击通知时执行的操作（打开 MainActivity）|
| `FLAG_IMMUTABLE` | PendingIntent 不可被修改（Android 12+ 要求）|
| `setSmallIcon()` | 通知栏显示的小图标 |
| `IMPORTANCE_LOW` | 不发出声音和震动 |

## Android 版本兼容处理

代码中多处使用版本检查来处理不同 Android 版本的差异：

```kotlin
// Android 13+ 需要通知权限
if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.TIRAMISU) {
    // 请求 POST_NOTIFICATIONS 权限
}

// Android 14+ 前台服务需要指定类型
if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.UPSIDE_DOWN_CAKE) {
    startForeground(NOTIFICATION_ID, notification,
        ServiceInfo.FOREGROUND_SERVICE_TYPE_MEDIA_PROJECTION)
} else {
    startForeground(NOTIFICATION_ID, notification)
}
```

| Android 版本 | API Level | 代码名称 | 相关变化 |
|-------------|-----------|---------|---------|
| Android 8.0 | 26 | Oreo | 通知渠道必须、前台服务要求 |
| Android 12 | 31 | S | PendingIntent 必须指定可变性 |
| Android 13 | 33 | Tiramisu | 通知权限需运行时申请 |
| Android 14 | 34 | UpsideDownCake | 前台服务必须在启动后才能创建 MediaProjection |

## 安全最佳实践

本项目遵循的安全原则：

1. **最小权限原则**
   - 只申请必需的权限
   - 无障碍服务只请求按键拦截，不请求其他能力

2. **`exported="false"`**
   - 非入口组件设为不对外暴露，防止其他 App 恶意调用

3. **`MODE_PRIVATE`**
   - SharedPreferences 使用私有模式，数据只有本 App 能访问

4. **及时清理敏感数据**
   - 截屏数据在使用后立即回收（`bitmap.recycle()`）
   - 服务销毁时清理所有资源

## 小结

| 要点 | 说明 |
|------|------|
| 普通权限 | 安装时自动授予（INTERNET 等）|
| 危险权限 | 需运行时弹窗申请（POST_NOTIFICATIONS）|
| 特殊权限 | 需跳转系统设置（SYSTEM_ALERT_WINDOW）|
| MediaProjection | 单独的截屏授权机制 |
| 无障碍服务 | 必须用户手动在系统设置中开启 |
| 前台服务通知 | Android 8.0+ 必须显示通知 |
| 版本兼容 | 不同 Android 版本有不同安全要求 |

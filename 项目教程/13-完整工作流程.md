# 🔄 完整工作流程 —— 从启动到翻译

## 串联整个 App 的运行逻辑

前面的教程分别讲解了每个文件的功能。现在我们把它们串联起来，看看从用户打开 App 到看到翻译结果，整个过程是怎么做到的。

---

## 阶段一：App 启动

### 用户点击桌面图标

```
Android 系统
    │
    │ 根据 AndroidManifest.xml 中的 intent-filter
    │ 找到 MAIN + LAUNCHER 的 Activity
    ↓
MainActivity.onCreate()
    │
    ├── 1. 加载布局 activity_main.xml → 通过 ViewBinding 绑定控件
    ├── 2. 创建 AppConfig 实例 → 读取 SharedPreferences 中已保存的配置
    ├── 3. 调用 updateUI() → 显示 API 状态和按钮文字
    ├── 4. 设置按钮点击事件
    └── 5. 设置透明度滑块监听
```

**用户看到的画面**：主界面，显示"开始翻译"按钮和 API 状态。

---

## 阶段二：配置 API（首次使用）

```
用户点击"设置"
    │
    ↓
SettingsActivity.onCreate()
    │
    ├── 加载 activity_settings.xml
    ├── 从 AppConfig 读取已有配置 → 填充到输入框
    └── 设置语言下拉菜单、按钮事件

    用户填写 API URL、API Key、模型名称
    选择源语言和目标语言
    │
    ↓
用户点击"保存"
    │
    ├── 验证输入（不能为空）
    ├── 调用 config.apiUrl = url 等 → SharedPreferences 保存
    ├── Toast 提示"设置已保存"
    └── finish() → 返回 MainActivity

    MainActivity.onResume()
    │
    └── updateUI() → 显示"✅ API 已配置: deepseek-chat"
```

**数据流向**：
```
用户输入 → SettingsActivity → AppConfig → SharedPreferences（本地文件）
```

## 阶段三：启动翻译服务

```
用户点击"开始翻译"
    │
    ↓
startOverlayFlow()
    │
    ├── [检查1] API 已配置？ ✅
    │
    ├── [检查2] 通知权限？
    │   └── Android 13+：弹出系统对话框 → 用户同意 → 返回 startOverlayFlow()
    │
    ├── [检查3] 悬浮窗权限？
    │   └── 跳转系统设置 → 用户打开开关 → 返回 → onActivityResult()
    │                                              └── requestScreenCapture()
    │
    └── [检查4] 截屏权限
        └── requestScreenCapture() → 弹出系统确认框 → 用户同意
                                                       │
                                                       ↓
                                            onActivityResult()
                                                       │
            ┌──────────────────────────────────────────┘
            │
            ├── 保存授权数据到 MainActivity 静态变量
            │   pendingResultCode = RESULT_OK
            │   pendingResultData = data
            │
            └── startOverlayService()
                    │
                    └── startForegroundService(Intent → OverlayService)
```

## 阶段四：悬浮窗服务启动

```
OverlayService.onCreate()
    │
    ├── instance = this（单例注册）
    ├── 初始化 AppConfig 和 TranslationApi
    ├── 获取屏幕尺寸和密度
    └── 计算悬浮窗初始大小

OverlayService.onStartCommand()
    │
    ├── 1. 创建通知渠道
    ├── 2. startForeground() → 在通知栏显示"实时翻译运行中"
    │
    ├── 3. 取出截屏授权数据
    │   ├── resultCode, resultData ← MainActivity 静态变量
    │   └── 清除静态变量（用完即弃）
    │
    ├── 4. 创建 MediaProjection
    │   ├── mediaProjection = projectionManager.getMediaProjection(...)
    │   ├── setupImageReader() → 创建 ImageReader + VirtualDisplay
    │   └── 开始持续接收屏幕画面帧 → 保存到 latestBitmap
    │
    └── 5. setupOverlay() → 创建悬浮窗
        ├── 加载 overlay_layout.xml → 翻译窗口
        ├── 设置 WindowManager.LayoutParams（位置、大小、标志）
        ├── windowManager.addView(overlayView) → 显示翻译窗口
        │
        ├── 加载 edit_button.xml → 编辑按钮
        ├── windowManager.addView(editFabView) → 显示编辑按钮
        │
        └── 设置触摸事件监听
```

**用户看到的画面**：屏幕上出现半透明黑色矩形窗口和一个蓝色圆形编辑按钮。

## 阶段五：执行翻译

### 方式一：点击悬浮窗

```
用户点击翻译窗口（非编辑模式下）
    │
    ↓
scrollView.onTouchListener → ACTION_UP → !isDragging
    │
    ↓
captureAndTranslate()
```

### 方式二：按快捷键

```
用户按下外接键盘绑定的翻译键
    │
    ↓
KeyEventService.onKeyEvent()
    ├── 获取 AppConfig → 读取绑定的 keyCode
    ├── 获取 OverlayService.getInstance()
    └── event.keyCode == translateKey → service.triggerTranslate()
                                            │
                                            ↓
                                    captureAndTranslate()
```

### `captureAndTranslate()` 完整流程

```
captureAndTranslate()
    │
    ├── 1. 显示"识别中..."和加载动画
    │
    ├── 2. 隐藏悬浮窗（避免截图包含自己）
    │       overlayView.visibility = INVISIBLE
    │       editFabView.visibility = INVISIBLE
    │
    ├── 3. 等待 200ms（让屏幕画面更新）
    │
    ├── 4. 恢复悬浮窗显示
    │
    ├── 5. 获取最新屏幕截图
    │       bitmap = latestBitmap.copy()
    │
    ├── 6. 裁剪悬浮窗区域
    │       cropBitmap = cropOverlayRegion(bitmap)
    │       │
    │       │  如果截图失败 → 显示错误信息 → 结束
    │       │
    ├── 7. OCR 文字识别
    │       performOcrWithBlocks(cropBitmap)
    │       │
    │       ├── 根据源语言选择识别器（中/日/韩/英文）
    │       ├── recognizer.process(image)
    │       └── 返回 List<OcrBlock>（文字 + 位置）
    │           │
    │           │  如果未识别到文字 → 显示"未识别到文字" → 结束
    │           │
    ├── 8. 构建翻译请求
    │       fullText = blocks.joinToString("\n||||\n") { it.text }
    │       │
    │       │  例如："Hello World\n||||\nGood Morning"
    │       │
    ├── 9. 调用翻译 API（在协程中）
    │       serviceScope.launch {
    │           result = translationApi.translate(fullText)
    │       }
    │       │
    │       ├── 构建 JSON 请求体（system prompt + user message）
    │       ├── OkHttp 发送 POST 请求 → API 服务器
    │       ├── 接收 JSON 响应 → 解析出翻译结果
    │       └── 返回 Result<String>
    │           │
    │           │  例如："你好世界\n||||\n早上好"
    │           │
    ├── 10. 拆分翻译结果
    │        translatedParts = translated.split("||||")
    │        │
    │        ├── "你好世界"   → 对应 blocks[0]
    │        └── "早上好"     → 对应 blocks[1]
    │
    ├── 11. 渲染翻译图片
    │        renderTranslatedBitmap(cropBitmap, blocks, translatedParts)
    │        │
    │        ├── 对每个文字块：
    │        │   ├── sampleBackgroundColor() → 采样周围背景色
    │        │   ├── canvas.drawRect() → 用背景色覆盖原文
    │        │   ├── fitTextSize() → 计算合适字号
    │        │   ├── getContrastColor() → 选择黑或白文字色
    │        │   └── drawTextInBox() → 绘制翻译文字（自动换行）
    │        └── 返回翻译后的 Bitmap
    │
    └── 12. 显示结果
             ivTranslation.setImageBitmap(resultBitmap)
             ivTranslation.visibility = VISIBLE
             scrollView.visibility = GONE
```

## 阶段六：清除翻译 / 关闭服务

### 清除翻译结果

```
用户点击窗口外部 / 按清除快捷键
    │
    ↓
doClearTranslation()
    ├── tvTranslation.text = ""
    ├── ivTranslation.visibility = GONE（隐藏翻译图片）
    ├── scrollView.visibility = VISIBLE（恢复文字区域）
    └── applyOverlayAlpha()（恢复半透明背景）
```

### 关闭服务

```
方式1：在 MainActivity 点击"停止翻译"
    └── stopOverlayService() → OverlayService.stopService()

方式2：长按编辑按钮
    └── editFabView.setOnLongClickListener → stopSelf()

OverlayService.onDestroy()
    │
    ├── instance = null（清除单例引用）
    ├── windowManager.removeView(overlayView)（移除翻译窗口）
    ├── windowManager.removeView(editFabView)（移除编辑按钮）
    ├── cleanupProjection()
    │   ├── virtualDisplay.release()
    │   ├── imageReader.close()
    │   └── latestBitmap.recycle()
    ├── mediaProjection.stop()
    └── serviceScope.cancel()（取消所有协程）
```

## 数据流总结

```
屏幕内容
    │ MediaProjection + VirtualDisplay
    ↓
Bitmap（屏幕截图）
    │ cropOverlayRegion()
    ↓
Bitmap（窗口区域裁剪）
    │ ML Kit OCR
    ↓
List<OcrBlock>（文字 + 位置）
    │ joinToString("||||")
    ↓
String（待翻译文本）
    │ OkHttp → API Server → JSON
    ↓
String（翻译结果）
    │ split("||||")
    ↓
List<String>（翻译后的文字块）
    │ Canvas 绘制
    ↓
Bitmap（翻译覆盖图）
    │ ImageView 显示
    ↓
用户看到翻译结果
```

## 线程模型

```
主线程（UI Thread）                    IO 线程
    │                                   │
    ├── UI 操作                          │
    ├── 触摸事件处理                      │
    ├── 悬浮窗更新                       │
    ├── OCR 回调处理                     │
    │                                   │
    │   serviceScope.launch {           │
    │       withContext(IO) { ──────→   翻译 API 网络请求
    │       }            ←─────────    返回结果
    │       更新 UI                     │
    │   }                              │
    │                                   │
    │   handler.post { ... }           │
    │   （确保在主线程执行）               │
    │                                   │

ImageReader 回调线程
    │
    └── 处理屏幕帧 → 更新 latestBitmap
```

**关键规则**：
- **UI 操作必须在主线程**（更新控件、显示 Toast 等）
- **网络请求必须在非主线程**（否则会导致 ANR —— 应用无响应）
- 协程 + `Dispatchers.IO` 自动管理线程切换

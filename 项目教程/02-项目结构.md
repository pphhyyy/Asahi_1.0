# 📂 Android 项目结构详解

## 什么是 Android 项目结构？

一个 Android 项目不是随便把代码文件放在一起就行的。它有一套**固定的目录结构**，Gradle 构建工具会按照这个结构去编译、打包你的 App。

---

## 本项目的完整目录结构

```
translator_2.0/                      ← 项目根目录
├── build.gradle.kts                 ← 根级构建脚本（定义插件版本）
├── settings.gradle.kts              ← 项目设置（定义包含哪些模块）
├── gradle.properties                ← Gradle 全局属性

│   │
│   └── src/                         ← 源代码目录
```

## 逐个目录/文件解释

### 🔹 根目录文件

#### `build.gradle.kts`（根级）
定义整个项目使用的**插件版本**。注意它和 `app/build.gradle.kts` 是不同的文件。

```kotlin
plugins {
    id("com.android.application") version "8.2.0" apply false
    id("org.jetbrains.kotlin.android") version "1.9.22" apply false
}
```

- `com.android.application`：Android 应用插件，版本 8.2.0
- `org.jetbrains.kotlin.android`：Kotlin Android 插件，版本 1.9.22
- `apply false`：表示在根项目中只声明版本，不实际应用（由子模块 `app` 来应用）

#### `settings.gradle.kts`
定义项目的名称和包含的模块。

```kotlin
rootProject.name = "Translator"   // 项目名
include(":app")                   // 包含 app 模块
```

Android 项目可以有多个模块（如 app、library 等），这里只有一个 `app` 模块。

#### `gradle.properties`
Gradle 全局配置属性。

```properties
android.useAndroidX=true          # 使用 AndroidX（新版支持库）
kotlin.code.style=official        # Kotlin 代码风格
org.gradle.jvmargs=-Xmx2048m     # 给 Gradle 分配的最大内存
```

#### `gradlew` / `gradlew.bat`
**Gradle Wrapper**（Gradle 包装器）脚本。它的作用是：即使你电脑上没有安装 Gradle，这个脚本也能自动下载指定版本的 Gradle 来编译项目。

- `gradlew`：macOS/Linux 用
- `gradlew.bat`：Windows 用

---

### 🔹 `gradle/wrapper/` 目录

#### `gradle-wrapper.properties`
指定项目使用的 Gradle 版本。

```properties
distributionUrl=https\://services.gradle.org/distributions/gradle-8.5-bin.zip
```

这确保了所有开发者使用**完全相同**的 Gradle 版本，避免"在我电脑上能跑"的问题。

---

### 🔹 `app/` 目录 —— 应用模块

这是你的应用代码所在的地方，也是你最常编辑的目录。

#### `app/build.gradle.kts`（模块级）
定义这个模块的编译配置、依赖库等。这是项目中**最重要的配置文件之一**，我们会在下一篇教程中详细讲解。

#### `app/proguard-rules.pro`
代码混淆规则文件。在发布(release)版本时，ProGuard 可以混淆和压缩你的代码，防止被反编译。本项目暂未启用混淆（`isMinifyEnabled = false`）。

---

### 🔹 `app/src/main/` 目录 —— 源代码和资源

#### `AndroidManifest.xml`
App 的"**身份证**"。它告诉 Android 系统：
- 这个 App 叫什么名字
- 包含哪些 Activity（界面）
- 包含哪些 Service（服务）
- 需要哪些权限

这是每个 Android 项目**必须有的**文件，详见后续教程。

---

### 🔹 `app/src/main/java/` 目录 —— 代码文件

虽然文件夹叫 `java/`，但里面放的是 **Kotlin** 代码（`.kt` 文件）。这是历史原因——Android 最初只支持 Java，后来兼容了 Kotlin，但目录名没变。

代码按照**包名**组织：`com/translator/app/`

| 文件 | 职责 |
|------|------|
| `AppConfig.kt` | 管理 App 配置的读写 |
| `MainActivity.kt` | 主界面 Activity |
| `SettingsActivity.kt` | 设置界面 Activity |
| `KeyEventService.kt` | 无障碍按键监听服务 |
| `TranslationApi.kt` | 翻译 API 客户端 |
| `overlay/OverlayService.kt` | 悬浮窗核心服务 |

---

### 🔹 `app/src/main/res/` 目录 —— 资源文件

在 Android 中，**代码和资源是分离的**。界面布局、文字、颜色、图标等都放在 `res/` 目录下。

#### `res/layout/` —— 布局文件
使用 XML 描述界面的外观和结构。每个 Activity 或弹窗通常对应一个布局文件。

| 文件 | 用途 |
|------|------|
| `activity_main.xml` | 主界面布局 |
| `activity_settings.xml` | 设置界面布局 |
| `overlay_layout.xml` | 悬浮翻译窗口布局 |
| `floating_button.xml` | 悬浮翻译按钮布局 |
| `edit_button.xml` | 编辑模式按钮布局 |

#### `res/values/` —— 值资源

| 文件 | 用途 |
|------|------|
| `strings.xml` | 所有文字字符串（方便多语言翻译）|
| `colors.xml` | 颜色定义 |
| `themes.xml` | 主题和样式 |

> 💡 **为什么不直接在代码里写字符串？**
> 因为把字符串提取到 `strings.xml` 中，以后做多语言版本时只需要添加一个翻译文件，不需要改代码。这是 Android 的最佳实践。

#### `res/drawable/` —— 图形资源

| 文件 | 用途 |
|------|------|
| `fab_background.xml` | 悬浮按钮的圆形背景 |
| `overlay_background.xml` | 悬浮窗的矩形背景 |

这些是用 XML 定义的**矢量图形**（Shape Drawable），比图片更灵活。

#### `res/xml/` —— XML 配置文件

| 文件 | 用途 |
|------|------|
| `accessibility_service_config.xml` | 无障碍服务的配置 |

#### `res/mipmap-*/` —— 应用图标
存放不同分辨率的应用图标。`anydpi-v26` 表示使用**自适应图标**（Android 8.0+），系统会自动裁剪成圆形、方形等不同形状。

---

### 🔹 `build/` 目录 —— 构建输出

这个目录是**自动生成的**，包含编译产物：
- 编译后的 `.class` 文件
- 生成的 R 类（资源引用）
- 打包好的 APK 文件
- ViewBinding 生成的代码

> ⚠️ **永远不要手动修改 build/ 目录里的文件**，它们会在下次编译时被覆盖。
> 如果遇到奇怪的编译问题，可以尝试 **Build → Clean Project** 清空这个目录。

## Android 项目的"三层结构"

理解 Android 项目的关键是认识到它有三个层次：

```
┌─────────────────────────────┐
│   根项目（Root Project）      │  ← build.gradle.kts (根级)
│                              │     settings.gradle.kts
│   ┌─────────────────────┐   │
│   │   app 模块（Module）  │   │  ← app/build.gradle.kts
│   │                      │   │
│   │   ┌──────────────┐  │   │
│   │   │ src/main     │  │   │  ← 你的代码和资源
│   │   │  ├── java/   │  │   │
│   │   │  ├── res/    │  │   │
│   │   │  └── Manifest│  │   │
│   │   └──────────────┘  │   │
│   └─────────────────────┘   │
└─────────────────────────────┘
```

---

## 小结

| 要点 | 说明 |
|------|------|
| 代码放在 `app/src/main/java/` | 虽然叫 java，也放 Kotlin |
| 资源放在 `app/src/main/res/` | 布局、字符串、颜色、图标等 |
| 配置在 `build.gradle.kts` | 有根级和模块级两个 |
| `AndroidManifest.xml` | App 的全局配置声明 |
| `build/` 目录 | 自动生成，不要手动改 |

# 🎨 布局与 UI 设计

## Android 的界面是怎么写的？

在 Android 中，界面使用 **XML 文件**来描述。你可以把 XML 布局想象成"界面的设计图纸"——它定义了界面上有哪些控件（按钮、文字、输入框等），以及它们的位置、大小、颜色。

布局文件统一放在 `app/src/main/res/layout/` 目录下。

---

## 基础概念：View 和 ViewGroup

在 Android 中，所有 UI 元素都是 **View**（视图）：

- **View**：基础控件，如 `TextView`（文字）、`Button`（按钮）、`EditText`（输入框）
- **ViewGroup**：容器，可以包含其他 View，如 `LinearLayout`（线性布局）、`FrameLayout`（帧布局）

它们的关系就像"盒子套盒子"：

```
ViewGroup（大盒子）
├── View（按钮）
├── View（文字）
└── ViewGroup（小盒子）
    ├── View（输入框）
    └── View（文字）
```

---

## 常用布局类型

| 布局 | 特点 | 适用场景 |
|------|------|---------|
| `LinearLayout` | 子元素按**一行**或**一列**排列 | 简单的列表式界面 |
| `FrameLayout` | 子元素**层叠**显示 | 覆盖叠加的场景 |
| `ScrollView` | 可**滚动**的容器 | 内容超出屏幕时 |
| `ConstraintLayout` | 用约束关系定位 | 复杂的界面布局 |

本项目主要使用了 `LinearLayout`、`FrameLayout` 和 `ScrollView`。

---

## 常用 XML 属性

在了解具体布局之前，先掌握最常用的 XML 属性：

### 尺寸属性
| 属性 | 含义 |
|------|------|
| `layout_width="match_parent"` | 宽度和父容器一样宽 |
| `layout_width="wrap_content"` | 宽度根据内容自适应 |
| `layout_width="200dp"` | 固定宽度 200dp |

### 间距属性
| 属性 | 含义 |
|------|------|
| `padding="16dp"` | 内边距（控件内部的空白）|
| `layout_margin="16dp"` | 外边距（控件外部的空白）|
| `layout_marginBottom="16dp"` | 仅底部外边距 |

> 💡 **dp 是什么？**
> dp（density-independent pixels，密度无关像素）是 Android 的尺寸单位。不同手机屏幕密度不同，使用 dp 可以确保界面在不同设备上看起来大小一致。

### 文字属性
| 属性 | 含义 |
|------|------|
| `text="你好"` | 显示的文字 |
| `textSize="16sp"` | 文字大小（sp 是字体专用单位，会随系统字体大小设置变化）|
| `textColor="#FF0000"` | 文字颜色（十六进制 RGB）|
| `textStyle="bold"` | 文字加粗 |

## 本项目的 5 个布局文件

### 1. `activity_main.xml` —— 主界面

这是用户打开 App 首先看到的界面。

```xml
<LinearLayout
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical"
    android:padding="24dp"
    android:gravity="center">
```

- 使用**垂直线性布局**（`orientation="vertical"`），所有子元素从上到下排列
- `gravity="center"` 让内容居中

```
└──────────────────────────┘
```

#### 关键控件

```xml
<!-- API 状态提示 -->
<TextView
    android:id="@+id/tvApiStatus"
    android:text="⚠️ 请先配置 API"
    android:background="#FFF3E0" />
```

- `android:id="@+id/tvApiStatus"`：给控件设一个 ID，代码中可以通过这个 ID 找到它

```xml
<!-- 开始/停止按钮 -->
```

这里使用了 **Material Design 按钮**（`MaterialButton`），比普通 `Button` 更好看。

```xml
<!-- 透明度滑块 -->
<SeekBar
    android:id="@+id/seekBarAlpha"
    android:max="255"
    android:progress="128" />
```

`SeekBar` 是一个**滑动条控件**，用来调节悬浮窗的透明度（0~255）。

### 3. `overlay_layout.xml` —— 悬浮翻译窗口

这是显示在屏幕上方的悬浮翻译窗口。

```xml
<FrameLayout
    android:id="@+id/overlayRoot"
    android:background="@drawable/overlay_background">

    <!-- 翻译结果图片（图片替换模式） -->
    <ImageView android:id="@+id/ivTranslation" android:visibility="gone" />

    <!-- 翻译结果文字（fallback） -->
    <ScrollView android:id="@+id/scrollResult">
        <TextView android:id="@+id/tvTranslation" />
    </ScrollView>

    <!-- 加载指示器 -->
    <ProgressBar android:id="@+id/progressBar" android:visibility="gone" />

    <!-- 隐藏按钮 -->
    <TextView android:id="@+id/btnMinimize" android:text="−" />
</FrameLayout>
```

使用 `FrameLayout` 是因为翻译结果有两种显示模式：
- **图片模式**：`ImageView` 显示，`ScrollView` 隐藏
- **文字模式**：`ScrollView` 显示，`ImageView` 隐藏

它们**叠放**在一起，根据需要显示其中一个。

> `android:visibility="gone"` 表示控件**隐藏且不占空间**。另一个选项 `invisible` 是隐藏但仍占空间。

## Drawable 资源 —— 用 XML 画图形

### `fab_background.xml` —— 圆形按钮背景

```xml
<shape android:shape="oval">
    <solid android:color="#E01976D2" />
    <stroke android:width="1dp" android:color="#40FFFFFF" />
</shape>
```

| 标签 | 含义 |
|------|------|
| `shape="oval"` | 椭圆形（正方形的椭圆 = 圆形）|
| `solid` | 填充颜色 `#E01976D2`（半透明蓝色）|
| `stroke` | 描边——1dp 宽的半透明白色边框 |

> 💡 颜色 `#E01976D2` 中，`E0` 是透明度（0~FF），`1976D2` 是蓝色 RGB。

### `overlay_background.xml` —— 悬浮窗背景

```xml
<shape android:shape="rectangle">
    <solid android:color="#CC000000" />
    <corners android:radius="8dp" />
    <stroke android:width="1dp" android:color="#44FFFFFF" />
</shape>
```

- 矩形，8dp 圆角
- `#CC000000`：80% 不透明的黑色背景
- 半透明白色边框

```xml
    <color name="primary_dark">#1565C0</color>  <!-- 深蓝 -->
```

### `themes.xml` —— 主题样式

```xml
<style name="Theme.Translator" parent="Theme.MaterialComponents.Light.NoActionBar">
    <item name="colorPrimary">@color/primary</item>
```

- 继承自 `Theme.MaterialComponents.Light.NoActionBar`（Material Design 亮色主题，不带顶部操作栏）
- 自定义了三个颜色：主色调、深色主色调、强调色

## ViewBinding —— 代码中如何访问布局控件

在代码中访问布局控件有两种方式：

### 旧方式：`findViewById()`

```kotlin
val button = findViewById<Button>(R.id.btnToggle)
button.text = "点我"
```

缺点：需要手动指定类型，容易出错。

### 新方式：ViewBinding（本项目使用）

```kotlin
// 在 Activity 中
private lateinit var binding: ActivityMainBinding

override fun onCreate(savedInstanceState: Bundle?) {
    binding = ActivityMainBinding.inflate(layoutInflater)
    setContentView(binding.root)
    
    binding.btnToggle.text = "点我"  // 直接通过 binding 访问
    binding.tvApiStatus.text = "API 已配置"
}
```

ViewBinding 会为每个布局文件自动生成一个绑定类：
- `activity_main.xml` → `ActivityMainBinding`
- `activity_settings.xml` → `ActivitySettingsBinding`

优点：
- **类型安全**：不会把 `Button` 误当成 `TextView`
- **空安全**：如果布局中没有这个控件，编译时就会报错，而不是运行时崩溃
- **简洁**：不需要写 `findViewById()`

## 小结

| 要点 | 说明 |
|------|------|
| 布局用 XML 描述 | 放在 `res/layout/` 目录 |
| View = 控件 | TextView、Button、EditText 等 |
| ViewGroup = 容器 | LinearLayout、FrameLayout 等 |
| dp = 尺寸单位 | 屏幕密度无关，保证一致性 |
| sp = 字体单位 | 会跟随系统字体设置 |
| Drawable = XML 图形 | 可以用代码画圆形、矩形等 |
| ViewBinding | 安全、便捷地访问控件 |

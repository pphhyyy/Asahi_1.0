# 📱 实时翻译 App —— 项目概述

## 这个 App 是什么？

**实时翻译（Translator 2.0）** 是一个 Android 悬浮窗翻译工具。它的核心功能是：

1. 在屏幕上显示一个**半透明的悬浮窗口**
2. 对悬浮窗口覆盖区域的屏幕内容进行**截图**
3. 使用 **OCR（光学字符识别）** 技术识别截图中的文字
4. 调用**大语言模型 API**（如 DeepSeek、OpenAI）将文字翻译成目标语言
5. 将翻译结果**直接覆盖显示**在原文字的位置上

简单来说，它就像一个"屏幕实时翻译器"——你把窗口拖到任何有外语文字的地方，点一下就翻译。

---

## 核心功能一览

| 功能 | 说明 |
|------|------|
| 🪟 悬浮窗 | 可拖动、可调整大小的系统级悬浮窗口 |
| 📸 屏幕截图 | 使用 MediaProjection API 截取屏幕内容 |
| 🔍 OCR 文字识别 | 使用 Google ML Kit，支持中/英/日/韩文 |
| 🌐 AI 翻译 | 调用兼容 OpenAI 格式的大模型 API |
| 🖼️ 图片替换翻译 | 在原图上用背景色覆盖原文，写上译文 |
| ⌨️ 快捷键绑定 | 通过无障碍服务支持外接键盘/手柄按键触发 |
| ⚙️ 可配置 | API 地址、密钥、模型、语言对 均可自定义 |

## 使用的技术栈

### 开发语言
- **Kotlin**：Android 官方推荐的开发语言（比 Java 更简洁现代）

### Android 核心技术
| 技术 | 用途 |
|------|------|
| **ViewBinding** | 安全地访问布局中的 UI 控件 |
| **Service（前台服务）** | 在后台运行悬浮窗和截屏功能 |
| **WindowManager** | 创建和管理系统级悬浮窗口 |
| **MediaProjection** | 获取屏幕截图权限并截取屏幕 |
| **AccessibilityService** | 拦截外接键盘按键事件 |
| **SharedPreferences** | 本地持久化存储配置信息 |
| **Coroutines（协程）** | 处理异步操作（网络请求、OCR）|

### 第三方库
| 库 | 用途 |
|---|------|
| **Google ML Kit** | 本地 OCR 文字识别（无需联网）|
| **OkHttp** | 发送 HTTP 网络请求（调用翻译 API）|
| **Gson** | JSON 数据的序列化和反序列化 |
| **Material Components** | Google 官方 UI 组件库 |

## App 的工作流程（简化版）

```
用户点击"开始翻译"
    ↓
申请权限（悬浮窗 + 截屏 + 通知）
    ↓
启动前台服务 → 显示悬浮窗 + 浮动按钮
    ↓
用户拖动窗口到需要翻译的位置
    ↓
点击悬浮窗（或按快捷键）
    ↓
隐藏悬浮窗 → 截取屏幕 → 裁剪窗口区域
    ↓
ML Kit OCR 识别文字（含位置信息）
    ↓
将识别的文字发送给大模型 API 翻译
    ↓
在截图上用背景色覆盖原文，绘制译文
    ↓
将翻译后的图片显示在悬浮窗中
```

## 项目的 6 个核心文件

| 文件 | 职责 | 类比 |
|------|------|------|
| `MainActivity.kt` | 主界面，启动/停止服务 | "遥控器" |
| `SettingsActivity.kt` | 设置界面，配置 API 和按键 | "设置面板" |
| `AppConfig.kt` | 读写配置（API Key、语言等）| "记事本" |
| `OverlayService.kt` | 悬浮窗 + 截屏 + OCR + 渲染 | "核心引擎" |
| `TranslationApi.kt` | 调用大模型 API 翻译文字 | "翻译官" |
| `KeyEventService.kt` | 拦截键盘按键触发翻译 | "快捷键监听器" |

# 🛠️ 开发环境搭建

## 你需要准备什么？

在开始开发 Android 应用之前，你需要安装以下工具：

---

## 1. 安装 Android Studio

**Android Studio** 是 Google 官方提供的 Android 开发 IDE（集成开发环境），它基于 IntelliJ IDEA，内置了编写代码、设计界面、编译运行、调试等所有你需要的功能。

### 下载地址
- 官网：https://developer.android.com/studio
- 选择你操作系统对应的版本（Windows / macOS / Linux）

### 安装步骤（以 macOS 为例）
1. 下载 `.dmg` 文件
2. 打开后将 Android Studio 拖入 Applications 文件夹
3. 首次启动会引导你完成初始化设置

### 首次启动的初始化配置
Android Studio 第一次打开时会自动下载必需组件：

1. **Android SDK**（软件开发工具包）—— 包含编译 Android 应用所需的所有工具
2. **Android SDK Platform**（平台工具）—— 对应特定 Android 版本的 API
3. **Build Tools**（构建工具）—— 用来编译和打包你的 App

> ⚠️ 初次下载可能需要较长时间（几 GB），请确保网络通畅。如果下载慢，可以配置代理或使用国内镜像源。

---

## 2. 配置 JDK（Java Development Kit）

本项目使用 **Java 17**。好消息是，Android Studio 内置了 JDK，通常不需要额外安装。

你可以在 Android Studio 中确认：
- 打开 **File → Project Structure → SDK Location**
- 查看 "JDK location" 是否已设置

如果需要手动设置，在 `app/build.gradle.kts` 中定义了：
```kotlin
compileOptions {
    sourceCompatibility = JavaVersion.VERSION_17
    targetCompatibility = JavaVersion.VERSION_17
}

kotlinOptions {
    jvmTarget = "17"
}
```

这意味着项目要求 Java 17 兼容级别。

---

## 3. 配置 Android SDK 版本

打开 Android Studio 的 **SDK Manager**（工具栏 → Tools → SDK Manager），确保安装了以下内容：

### SDK Platforms（平台）
| 名称 | API Level | 说明 |
|------|-----------|------|
| Android 14 (UpsideDownCake) | API 34 | 本项目的目标版本（targetSdk = 34）|
| Android 8.0 (Oreo) | API 26 | 本项目的最低支持版本（minSdk = 26）|

### SDK Tools（工具）
确保勾选以下工具：
- Android SDK Build-Tools
- Android SDK Platform-Tools
- Android Emulator（如果你用模拟器测试）

---

## 4. 配置 Gradle

**Gradle** 是 Android 项目的构建工具，负责编译代码、管理依赖、打包 APK。

本项目的 Gradle 配置：
- **Gradle 版本**：8.5
- **Android Gradle Plugin 版本**：8.2.0

这些在项目文件中已经定义好了，Android Studio 会自动下载。具体配置在：
- `gradle/wrapper/gradle-wrapper.properties` → 定义 Gradle 版本
- `build.gradle.kts`（根目录）→ 定义 Android Gradle Plugin 版本

> 💡 Android Studio 打开项目后，会自动执行 Gradle Sync（同步），下载所需依赖。如果同步失败，通常是网络问题。

---

## 5. 创建或导入项目

### 方式一：导入现有项目（推荐）
1. 打开 Android Studio
2. 选择 **File → Open**
3. 选择项目根目录 `translator_2.0`
4. 等待 Gradle Sync 完成

### 方式二：从零创建新项目
1. 打开 Android Studio
2. 选择 **New Project**
3. 选择 **Empty Activity** 模板
4. 填写项目信息：
   - Name: `Translator`
   - Package name: `com.translator.app`
   - Language: **Kotlin**
   - Minimum SDK: **API 26 (Android 8.0)**
5. 点击 **Finish**

---

## 6. 准备测试设备

### 方式一：使用真机（推荐）
此 App 使用了截屏和悬浮窗功能，**强烈建议使用真机测试**。

开启手机的开发者选项：
1. 打开手机 **设置 → 关于手机**
2. 连续点击 **版本号** 7 次
3. 返回设置，找到 **开发者选项**
4. 开启 **USB 调试**
5. 用 USB 数据线连接电脑
6. 手机弹出对话框时点击 **允许**

### 方式二：使用模拟器
在 Android Studio 中：
1. 打开 **Device Manager**（右侧边栏或 Tools → Device Manager）
2. 点击 **Create Device**
3. 选择一个手机型号（如 Pixel 6）
4. 选择系统镜像 **API 34**
5. 完成创建后启动模拟器

> ⚠️ 注意：模拟器可能无法完美支持 MediaProjection（截屏）功能，建议用真机测试。

---

## 7. 运行项目

1. 确保 Gradle Sync 成功（右下角无报错）
2. 选择目标设备（工具栏的设备下拉菜单）
3. 点击绿色的 **▶ Run** 按钮（或按 `Shift + F10`）
4. 等待编译完成，App 会自动安装到设备上并启动

### 常见问题排查

| 问题 | 解决方案 |
|------|---------|
| Gradle Sync 失败 | 检查网络连接，尝试配置 Gradle 代理 |
| 找不到 SDK | 在 SDK Manager 中下载缺失的 API |
| 编译报错 | 检查 Java 版本是否为 17 |
| 手机未识别 | 检查 USB 调试是否开启，尝试换数据线 |
| 运行时闪退 | 查看 Logcat 日志排查原因 |

---

## 8. 认识 Android Studio 的主要界面

```
┌──────────────────────────────────────────────┐
│  工具栏：运行、调试、设备选择                    │
├──────┬───────────────────────────┬───────────┤
│      │                           │           │
│ 项目  │      代码编辑器             │  右侧     │
│ 文件  │   （写代码的地方）           │  面板     │
│ 树    │                           │           │
│      │                           │           │
├──────┴───────────────────────────┴───────────┤
│  底部面板：Logcat / Terminal / Build Output    │
└──────────────────────────────────────────────┘
```

### 常用面板
- **Project 面板**（左侧）：查看项目文件结构
- **Logcat**（底部）：查看 App 运行日志，调试必备
- **Terminal**（底部）：内置终端，可运行命令行
- **Build Output**（底部）：查看编译输出和错误信息
